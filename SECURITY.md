<h1 align="center">Security Policy</h1>

## Scope

This is a personal dotfiles repository — a curated collection of configurations
and scripts for my own daily use. While it's public for reference and sharing,
it is **not intended as a production-grade product**.

That said, security still matters. If you spot something off, I'd love to hear
about it.

## Supported Versions

Only the latest commit on the `main` branch is actively maintained.

| Version    | Supported          |
| ---------- | ------------------ |
| `main`     | :white_check_mark: |
| Older refs | :x:                |

## Reporting a Vulnerability

If you discover a security issue — such as:

- Hardcoded secrets, tokens, or credentials
- Unsafe shell scripts or command injection risks
- Misconfigured permissions or exposed sensitive paths
- Supply-chain concerns in referenced dependencies

Please **do not open a public issue**. Instead, report it privately via:

- 🔒 [GitHub Security Advisories](../../security/advisories/new) (preferred)
- 📧 Or reach out directly through my GitHub profile

### What to Expect

| Stage             | Timeline           |
| ----------------- | ------------------ |
| Initial response  | Within 72 hours    |
| Triage & analysis | Within 7 days      |
| Fix & disclosure  | Case-by-case basis |

Valid reports will be acknowledged in the commit history or release notes
(unless you prefer to remain anonymous).

## Responsible Disclosure

I follow a **coordinated disclosure** approach. Please give me reasonable time
to address the issue before going public. Thanks for helping keep this
repository — and anyone using it — safe. 🙏
