## 一、Alembic 是啥

Alembic 是一个 数据库迁移工具，通常和 SQLAlchemy 一起用。

👉 核心作用一句话： 
> **用版本控制的方式管理数据库结构（schema）变化**

## 二、核心概念
1️⃣ Migration（迁移脚本）

就是一条“数据库变更记录”，比如：

- 新增表
- 修改字段
- 删除索引

👉 本质：Python 文件

2️⃣ Revision（版本号）

每个 migration 都有唯一 ID：
```python
revision = '1975ea83b712'
down_revision = 'ae1027a6acf'
```
👉 形成一条链：
```bash
A -> B -> C -> D
```
3️⃣ upgrade / downgrade
```python
def upgrade():
    # 升级（往前走）
    
def downgrade():
    # 回滚（往后走）
```
👉 就是：
- upgrade：应用变更
- downgrade：撤销变更

4️⃣ autogenerate（自动生成）

自动生成迁移脚本（但不是100%靠谱）

## 三、安装 & 初始化
```bash
# 安装
pip install alembic psycopg2-binary psycopg2 # 这里已pgsql为例
# 在项目根目录初始化
alembic init alembic
```
会新增如下:
```plaintext
├── alembic.ini
└── alembic/
    ├── env.py
    ├── script.py.mako
    └── versions/
```

## 四、配置数据库连接
编辑alembic.ini
```ini
sqlalchemy.url = postgresql+psycopg2://user:password@localhost:5432/dbname
```
或者: 直接改 alembic/env.py
```python
import os
from logging.config import fileConfig
from pathlib import Path

from sqlalchemy import engine_from_config, pool
from sqlalchemy.engine.url import make_url
from dotenv import load_dotenv

from alembic import context
from src.manus.models import Base

env_file = Path(__file__).resolve().parent.parent / ".env"
load_dotenv(env_file)


config = context.config
database_url = os.getenv("SQLALCHEMY_DATABASE_URI")

if not database_url:
    raise ModuleNotFoundError("SQLALCHEMY_DATABASE_URI is not set. Did you forget to configure .env?")

sync_url = database_url.replace(
    "postgresql+asyncpg",
    "postgresql+psycopg2"
)

# 不在alembic.ini中硬编码, 通过环境变量读取再替换同步驱动
config.set_main_option("sqlalchemy.url", sync_url)
```

## 五、接入 SQLAlchemy Model
先准备一个Base类文件:
```python
# 假设路径为: app/models/base.py
from sqlalchemy.orm import declarative_base

# 定义基础ORM类, 让所有模型都继承这个类
Base = declarative_base()
```
接着在`env.py`中导入:
```python
from app.models import Base  # 新增导入

target_metadata = Base.metadata  # 找到target_metadata并修改
```

## 六、创建迁移
这里推荐自动生成方式, 手动不作记录

在项目根目录下输入如下命令, -m 为备注, 可不填写, 会在 ./alembic/versions/ 下生成一个版本记录python文件
```bash
alembic revision --autogenerate -m "create table demo"
```
> 8e5e7fb1f43d_create_table_demo.py

**注意:** 创建迁移之后最好review一下 迁移文件中的内容是否符合当下的操作

## 七、执行迁移
```bash
alembic upgrade head  # 升级到最新版本
alembic downgrade -1  # 回退到上一个版本
alembic downgrade 8e5e7fb1f43d   # 回退到指定版本
alembic downgrade base  # 回退到最初版本
alembic current  # 查看当前版本
alembic history  # 查看历史版本
```
